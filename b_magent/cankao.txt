为什么这种方案更合理？

如果你的目标是让 Agent 在执行任务过程中逐渐提升能力，其实没有必要同时引入 DPO、RL、Continual Learning 等多种优化方式。

仅使用 LoRA 有几个优势：

结构简单：整个系统只有一个可训练模块（LoRA Adapter），框架清晰。
训练稳定：采用成熟的 SFT 交叉熵损失即可，无需设计复杂的奖励函数或偏好优化。
参数高效：冻结基础模型，仅更新少量 LoRA 参数，训练成本低，适合在线或周期性增量学习。
论文表达更聚焦：创新点集中在"Agent 如何自主生成训练数据并触发自我进化"，而不是复杂的训练算法。

因此，整个系统可以统一描述为：

Task Execution → Multi-Agent Evaluation → Self-Reflection → Automatic Dataset Construction → LoRA Fine-tuning → Updated Agent

这样的框架既符合当前 Agent 自进化(Self-Evolving Agent)研究的思路，又避免了引入过多训练策略，使论文重点放在自主学习闭环而不是训练技巧上。
